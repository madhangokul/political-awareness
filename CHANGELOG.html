Title & Tone Pass — Self-Realization Framing
Update Tone

Refined browser <title> tags and homepage copy across all three pages. The previous titles leaned declarative ("The Honest Picture", "Real Governance Flaws") — language that argues toward a conclusion. Shift goal: every label should feel like an invitation to examine, not a verdict being handed down.

Rule to maintain: Titles and eyebrow copy should read as questions or orientations ("Examine the Idea", "Read the Full Record"), not as findings ("Real Flaws", "Hidden Costs"). The body content can be direct — the framing should never be.

Ongoing
Core Principle — Self-Realization, Not Persuasion
Tone

This is not a political campaign. It is not trying to move someone's vote. Every word should serve one purpose: giving the reader a clearer picture of what they already believe and why — so they can hold it more consciously, or revise it more honestly.

Test for any new copy: Does this sentence invite reflection, or does it push a conclusion? If it pushes — rewrite it as a question or a documented fact.
Ongoing
Technical Constraints — Pure HTML Only
Constraint

No build tools, no frameworks, no backend, no npm. Every page must open directly in a browser from the file system. External CDN links (Google Fonts, Chart.js) are acceptable as they are read-only and degrade gracefully.

CSS/JS rule: All styles and scripts live in <style> and <script> blocks in the same file. No external .css or .js files. This keeps each article fully portable.
Ongoing
Adding New Articles — Checklist
Process

Steps to follow when adding a new piece to this repository.

1. Create a new self-contained .html file.
2. Add an inline SVG eye favicon matching the page's background colour.
3. Set <title> in examination framing, not verdict framing.
4. Add a card to index.html with appropriate stripe colour (amber = philosophy, teal = data).
5. Add an entry to this changelog with the date, what changed, and any design decisions worth remembering.
